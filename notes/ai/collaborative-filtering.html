Collaborative Filtering

Serce systemów rekomendacyjnych — "ludzie którzy kupili X, kupili też Y".

Czym jest Collaborative Filtering?

Collaborative Filtering (CF) to metoda rekomendacji oparta na zachowaniu użytkowników. Nie analizuje cech produktów — tylko to, jacy użytkownicy co kupowali. Jeśli użytkownik A i B kupowali podobne rzeczy, a A kupił produkt X którego B jeszcze nie ma — system zasugeruje B produkt X.

Dwa podejścia

1. Memory-based (Item-to-Item / User-to-User)

Oblicza podobieństwo między produktami lub użytkownikami na podstawie macierzy interakcji. Najprostsze i najszybsze do wdrożenia. 

import numpy as np
from sklearn.metrics.pairwise import cosine_similarity

# Macierz użytkownicy × produkty (1 = kupił, 0 = nie)
user_item_matrix = np.array([
    [1, 1, 0, 1, 0],   # user 0: kupił produkty A, B, D
    [1, 0, 1, 1, 0],   # user 1: kupił A, C, D
    [0, 1, 1, 0, 1],   # user 2: kupił B, C, E
    [1, 1, 0, 0, 0],   # user 3: kupił A, B
])

# Podobieństwo między produktami (transpozycja)
item_similarity = cosine_similarity(user_item_matrix.T)

# Rekomendacja dla usera 3 — co podobne do tego co już ma?
user_basket = np.array([[1, 1, 0, 0, 0]])  # ma A i B
scores = item_similarity.T @ user_basket.T
scores[00, 1]] = -1  # ukryj to co już ma

# Najlepsze rekomendacje:
product_names = ["A", "B", "C", "D", "E"]
top_items = np.argsort(scores[0])[::-1]
for idx in top_items[:3]:
    print(f"  → {product_names[idx]} (score: {scores[0, idx]:.3f})")

Scenariusz: rekomendacje w koszyku

Dla sklepu internetowego, item-to-item CF jest najprostszym i najczęściej wystarczającym rozwiązaniem. Logika: użytkownik ma w koszyku produkty [X, Y] → znajdź produkty najbardziej powiązane z X i Y. 

from collections import defaultdict
import json

# Budowa macierzy współwystępowania z logów zamówień
# orders = [{"items": ["A", "B", "D"]}, {"items": ["A", "C"]}, ...]

co_occurrence = defaultdict(lambda: defaultdict(int))
item_count = defaultdict(int)

for order in orders:
    items = order["items"]
    for item in items:
        item_count[item] += 1
    for i in items:
        for j in items:
            if i != j:
                co_occurrence[i][j] += 1

# Normalizacja — Jaccard lub lift
def recommend(cart_items, top_n=5):
    scores = defaultdict(float)
    for item in cart_items:
        for related, count in co_occurrence[item].items():
            if related not in cart_items:
                # Lift = P(A,B) / (P(A) * P(B)) — korelacja niezależna od popularności
                lift = count / (item_count[item] * item_count[related] / len(orders))
                scores[related] += lift

    return sorted(scores.items(), key=lambda x: x[1], reverse=True)[:top_n]

# Użytkownik ma w koszyku laptop i myszkę:
print(recommend(["laptop", "myszka"]))
# → [('podkładka pod myszkę', 4.2), ('hub USB-C', 3.8), ...]

Jak mierzyć jakość rekomendacji?

  • Precision@K — ile z K zarekomendowanych produktów jest trafnych
  • Recall@K — jaki odsetek trafnych produktów znalazł się w top-K
  • NDCG@K — jak dobrze uporządkowane są rekomendacje (ważna pozycja)
  • Coverage — jaki odsetek katalogu jest rekomendowany
from sklearn.metrics import precision_score, recall_score

# Test na zbiorze historycznym
# Dla każdego użytkownika: ukryj ostatnią zakupioną rzecz, sprawdź czy jest w top-K rekomendacji

def precision_at_k(relevant, recommended, k):
    rec_top_k = set(recommended[:k])
    hits = len(rec_top_k & set(relevant))
    return hits / k

# Przykład: user kupił [A, B, C], system rekomendował [B, X, A, Y]
print(precision_at_k(["A", "B", "C"], ["B", "X", "A", "Y"], k=3))
# → 0.67 (2 z 3 trafnych)

Podsumowanie

Collaborative Filtering to najprostsza i najczęściej wystarczająca metoda rekomendacji. Item-to-item CF (memory-based) jest idealny na start — szybki, prosty, nie wymaga treningu modelu. Matrix Factorization daje lepsze rezultacje przy dużych zbiorach danych. W e-commerce często łączy się CF z content-based filtering dla hybrydowego podejścia.

Plusy

  • Nie wymaga cech produktów — tylko dane transakcyjne
  • Item-to-item jest błyskawiczny w inferencji (<1ms)
  • Prosty do implementacji i utrzymania
  • Działa od razu po zebraniu pierwszych danych

Minusy

  • Cold start — nowe produkty nie mają historii, więc nie są rekomendowane
  • Sparse matrix problem — przy dużym katalogu i mało transakcjach macierz jest pusta
  • Nie rozumie "dlaczego" — tylko statystyczne powiązania
  • Popular bias — popularne produkty są nadrekomendowane
