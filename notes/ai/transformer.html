Transformer & Self-Attention

Architektura, która zmieniła AI — od "Attention Is All You Need" (2017) do LLM.

Czym jest Transformer?

Transformer to architektura neuronowa zaproponowana w artykule "Attention Is All You Need" (Vaswani et al., 2017). Zastąpił RNN/LSTM i umożliwił szkolenie modeli na bardzo długich sekwencjach dzięki mechanizmowi self-attention. 

# Self-Attention w jednym zdaniu:
# Dla każdego tokena obliczamy, jak ważny jest każdy inny token w sekwencji.

Jak działa Self-Attention?

Każdy token w sekwencji generuje trzy wektory: Query, Key i Value. Query jednego tokenu porównuje się z Key każdego innego tokonu (iloczyn macierzy) — wynik to "waga uwagi". Wagi mnożą Value, a wyniki sumują. 

# Self-Attention: Attention(Q, K, V) = softmax(Q @ K^T / sqrt(d_k)) @ V

Kluczowe właściwości:

  • Skalowanie 1/dk — dzielenie przez pierwiastek wymiaru zapobiega saturacji softmax (gradienty nie znikają)
  • Multi-Head Attention — kilka równoległych attention "nagłówków" pozwala modelowi skupić się na różnych aspektach relacji
  • Positional Encoding — Transformer nie ma pojęcia o kolejności tokenów, więc dodaje informacje o pozycji sinusoidalnie (lub uczy je)

Encoder vs Decoder-only

Transformer miał dwie części — encoder i decoder. Współczesne LLM używają tylko decoder-only:

  • Encoder (BERT) — widzi cały kontekst naraz → klasyfikacja, NER, ekstrakcja encji
  • Decoder-only (GPT-style) — generuje token po tokenie z lewą maską → LLM, kod, czaty

Implementacja Self-Attention w PyTorch

import torch
import torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model=512, num_heads=8):
        super().__init__()
        self.d_model = d_model
        self.num_heads = num_heads
        self.head_dim = d_model // num_heads

        # Linear projections dla Q, K, V i outputu
        self.q_proj = nn.Linear(d_model, d_model)
        self.k_proj = nn.Linear(d_model, d_model)
        self.v_proj = nn.Linear(d_model, d_model)
        self.out_proj = nn.Linear(d_model, d_model)

    def forward(self, x):  # x: (batch_size, seq_len, d_model)
        B, L, D = x.shape

        # Projekcje do Q, K, V
        q = self.q_proj(x).view(B, L, self.num_heads, self.head_dim)
        k = self.k_proj(x).view(B, L, self.num_heads, self.head_dim)
        v = self.v_proj(x).view(B, L, self.num_heads, self.head_dim)

        # Transpose dla attention: (batch, heads, seq_len, head_dim)
        q, k, v = q.transpose(1,2), k.transpose(1,2), v.transpose(1,2)

        # Scaled dot-product attention
        scores = q @ k.transpose(-2, -1) / (self.head_dim ** 0.5)  # (B, H, L, L)
        attn_weights = torch.softmax(scores, dim=-1)

        output = attn_weights @ v  # (B, H, L, head_dim)
        output = output.transpose(1,2).reshape(B, L, D)  # z powrotem do original shape

        return self.out_proj(output)

Kiedy Transformer?

  • NLP — tłumaczenie, generowanie tekstu, klasyfikacja, ekstrahcja encji
  • CSS — kod, opisy, dokumentacja (LLM: GPT, Llama, Mistral, Qwen)
  • Wizja komputerowa — ViT (Vision Transformer) do klasyfikacji obrazów
  • Multimodalne — multimodalne modele jak CLIP, LLaVA łączą tekst i obrazy

Podsumowanie

Transformer to architekturna, która zdominowała AI po 2017. Self-attention pozwala każdemu tokenowi "widzieć" każdy inny bezpośrednio — bez ograniczeń RNN do sekwencji liniowych. Decoder-only transformers (GPT-style) są standardem dla LLM.

Plusy

  • Długi kontekst — model "widzi" wszystkie tokeny naraz
  • Równoległość treningowa — brak sekwencyjności RNN = szybszy trening na GPU
  • Kompozycyjność — można łączyć z innymi architekturami (CNN, RNN, MoE)
  • Skalowalność — modele rosną od milionów do bilionów parametrów

Minusy

  • O(n²) złożoność attention (problem przy długich kontekstach)
  • Duże zapotrzebowanie na pamięć (mikrobatch size musi być mały)
  • Słaba interpretowalność — trudno zrozumieć co decyduje o output
