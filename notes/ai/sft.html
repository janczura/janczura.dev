SFT — Supervised Fine-Tuning

Dookrąglenie modelu nadzorowane na własnych danych.

Czym jest SFT?

Supervised Fine-Tuning to proces adaptacji gotowego, wstępnie wytrenowanego modelu (np. Llama, Mistral, CodeLlama) na własnym zbiorze danych z etykietami. Model już zna język lub kod — SFT uczy go konkretnego stylu, formatu lub domeny.

Dane są przygotowane jako pary (prompt → odpowiedź). W przypadku kodu: kontekst pliku + komentarz → poprawny fragment kodu w stylu projektu. Model uczy się kontynuować tekst zgodnie z wzorcem.

Jak to działa?

  1. Model bazowy — pobierasz gotowe LLM (np. CodeLlama-7b, Mistral-7b)
  2. Dataset — przygotowujesz pary JSONL: {"text": "<prompt>\n<response>"}
  3. Trening — model uczy się przewidywać odpowiedź po prompcie, całe wagі są aktualizowane
  4. Ewaluacja — sprawdzasz na zbiorze testowym czy model generuje poprawne odpowiedzi

Przykład — SFT z HuggingFace TRL

Poniżej pełny przykład fine-tuningu modelu CodeLlama na własnym kodzie:

from trl import SFTTrainer
from transformers import AutoModelForCausalLM, AutoTokenizer
from datasets import load_dataset

model_name = "codellama/CodeLlama-7b-hf"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype=torch.float16,
    device_map="auto"
)

# Dataset w formacie JSONL: {"text": "prompt\nresponse"}
dataset = load_dataset("json", data_files="./code_examples.jsonl")["train"]

trainer = SFTTrainer(
    model=model,
    tokenizer=tokenizer,
    train_dataset=dataset,
    dataset_text_field="text",
    max_seq_length=2048,
    args=TrainingArguments(
        output_dir="./sft-output",
        num_train_epochs=3,
        per_device_train_batch_size=2,
        gradient_accumulation_steps=8,
        learning_rate=2e-5,
        fp16=True,
        logging_steps=10,
    ),
)

trainer.train()

Przygotowanie danych — code_examples.jsonl

{  "text": "<<SYS>>\nPisz kod w stylu naszego projektu.\n<</SYS>>\n\nQ: Stwórz endpoint GET /users\nA:\n@app.route('/users', methods=['GET'])\ndef get_users():\n    return jsonify(User.query.all())"}

Scenariusz: styl kodu repozytorium

Aby przyuczyć model do pisania w stylu Waszego projektu, parsujesz repo i generujesz pary treningowe: 

import os
from pathlib import Path

repo_path = Path("./my-project")
examples = []

for py_file in repo_path.glob("**/*.py"):
    content = py_file.read_text()
    lines = content.split("\n")

    # Bierzemy kontekst (pierwsze 20 linii) → reszta jako odpowiedź
    context = "\n".join(lines[:20])
    response = "\n".join(lines[20:])

    examples.append({
        "text": f"<<SYS>>\nPisz kod w stylu projektu.\n<</SYS>>\n\n{context}\n{response}"
    })

# Zapisz do JSONL
with open("code_examples.jsonl", "w") as f:
    for ex in examples:
        f.write(json.dumps(ex) + "\n")

Podsumowanie

SFT to najprostsza i najbardziej bezpośrednia metoda adaptacji modelu. Działa na parze prompt→odpowiedź, aktualizując wszystkie wagі modelu. Idealne gdy masz solidny zbiór danych i chcesz żeby model "mówił" w konkretnym stylu lub specjalizował się w danej domenie.

Plusy

  • Proste w implementacji — jeden pipeline, gotowe biblioteki (TRL)
  • Działa na każdym modelu bazowym
  • Efektywność rośnie z jakością danych
  • Pełna kontrola nad zachowaniem modelu

Minusy

  • Wymaga dużo pamięci VRAM (cały model w RAM GPU)
  • Długi czas treningu przy dużych modelach
  • Ryzyko catastrophic forgetting — model może zapomnieć ogólną wiedzę
  • Każda nowa adaptacja wymaga pełnego re-treningu
