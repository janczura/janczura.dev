Model Evaluation Metrics

Jak mierzyć, czy model jest dobry.

Metryki dla modeli generatywnych

Dla LLM nie ma jednej "najlepszej" metryki — różne metryki mierzą różne aspekty. Oto najważniejsze:

Perplexity (Perypleksja)

Mierzi jak bardzo model jest "zaskoczony" tekstem. Niska perplexity = model dobrze zna ten język i styl. Obliczana na zbiorze walidacyjnym: 

# Perplexity = exp(-(1/N) · Σ log p(w_i | w_0 ... w_{i-1}))
  • Poniżej 20 — model "zna" język
  • Niższa perplexity na tym samym zbiorze = lepszy model (ale nie zawsze)

BLEU i ROUGE

Metryki oparte o n-gramy — porównują odpowiedź modelu z etykietą referencyjną:

  • BLEU (2004) — tłumaczenia maszynowe, tekst techniczny
  • ROUGE (2004) — sumaryzacja tekstu, LLMs
# Przykład BLEU: odpowiedź = "kot je rybę", referencja = "kot zjadł ryby"
# 1-gram overlap: kot(1/1), je/zjadł(0/1), rybę/ryby(0/1) → BLEU = 33%

Benchmarki

Poważne benchmarki używają dużych zbiorów i ręcznych ocen:

  • MMLU — 57 przedmiotów, od prawa do fizyki (knowledge)
  • HellaSwag / ARC — rozumowanie logiczne
  • SWE-bench — rozwiązywanie problemów w repozytoriach GitHub

Ocena ludzka (Human Eval)

Ostatecznie najlepsza metryka to ocena człowieka. Wymaga zestawu pytań i ocen "A vs B" — dokładnie to, czego używa RLHF/DPO. Można zrobić samemu lub przez platformy zewnętrzne (Scale AI, Appen). 

def human_eval(samples):
    # samples = [{"prompt": ..., "model_a": ..., "model_b": ...}]
    scores = []
    for sample in samples:
        winner = ask_human(
            prompt=sample["prompt"],
            answers=[sample["model_a"], sample["model_b"]],
        )
        scores.append(winner)  # "a", "b", lub "tie"

    return {"win_rate_a": sum(s=="a") / len(scores)}

Podsumowanie

Nie ma jednej metryki — używaj kilku. BLEU/ROUGE dla generatywnego tekstu, perplexity do monitorowania jakości języka, benchmarki (MMLU) i ocena ludzka do porównań modeli. Ostatecznie: najlepsza metryka to "czy użytkownik jest zadowolony".

Kluczowe wnioski

  • Benchmarki są lepsze niż metryki n-gramowe (BLEU często kłamie)
  • Ocena ludzka jest jedyną miarodajną dla finalnego produktu
  • Perplexity dobrze koreluje z jakością na tym samym domenie

Ograniczenia

  • Benchmarki szybko się "wyczerpują" — modele uczą się na datasetach testowych
  • Ocena ludzka jest droga i trudna do skalowania
  • Metryki automatyczne nie zawsze korelują z jakością postrzeganą przez ludzi
