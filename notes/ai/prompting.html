Prompt Engineering & Prompting Techniques

Jak mówić z modelem, by dostawać lepsze odpowiedzi.

Czym jest Prompt Engineering?

To sztuka formułowania promptów tak, by model wygenerował najlepszą możliwą odpowiedź. Dobre prompty mogą zmienić jakość odpowiedzi od "słabej" do "wyśmienitej" — i to bez fine-tuningu modelu.

Prompting: Few-Shot vs Zero-Shot

# Zero-shot (bez przykładów)
"Klasyfikuj sentiment: 'Ten produkt jest straszny' →"

# Few-shot (z 3-5 przykładami)
"Tekst: 'Uwielbiam ten film!' → Sentiment: Pozytywny
Tekst: 'Niestety, produkt zepsuty przy pierwszym użyciu' → Sentiment: Negatywny
Tekst: 'Ten produkt jest straszny' →"

Chain of Thought (CoT)

"Pomyśl krok po kroku" — nakłaniasz model do pokazania rozumowania przed odpowiedzią. Dramatycznie poprawia wyniki w zadaniach logicznych i matematycznych: 

Q: W szafie mam 5 czerwonych skarpet, 4 niebieskie i 3 zielone. Ile muszę wylosować na ciemno,
   żeby mieć pewność, że mam parę?
# CoT — model pokaże rozumowanie:
A: Najgorszy scenariusz — wyciągam po jednej z każdego koloru (3 skarpetki, brak pary).
   Następna musi się powtórzyć. Odpowiedź: 4.

Tree of Thoughts (ToT)

Rozszerzenie CoT — model generuje kilka możliwych rozwiązań, ocenia je, rozszerza te najlepsze i tak dalej. Przydatne przy złożonych problemach decyzyjnych: 

# ToT w pseudokodzie:
for each path in generate_paths(model, prompt, depth=3):
    score = evaluate(path)  # model sam ocenia swoją odpowiedź
    if score < min_threshold:
        prune(path)       # odrzucamy słabe ścieżki
return best_path_by_score()

ReAct — Reasoning + Acting

Model naprzemiennie myśli (Reason) i wykonuje akcje (Action): wyciąga narzędzia, szuka informacji. Kluczowe do RAG: 

# ReAct format:
Thought 1: Muszę znaleźć aktualny kurs USD/PLN.
Action: search("USD PLN exchange rate today")
Observation: [wynik wyszukiwania]
Thought 2: Mam dane — mogę obliczyć przeliczenie.
Answer: $100 = ~385 PLN (kurs z dnia dzisiejszego).

Najlepsze praktyki

  • Precyzja > długość — krótkie, konkretne prompty działają lepiej niż długie opisy
  • Dawaj kontekst i role — "Jesteś senior developerem w Pythonie" daje lepsze odpowiedzi niż brak roli
  • Pokazuj format wyjścia — "Odpowiedz jako JSON z polami: ..."
  • Few-shot > zero-shot — nawet 3 przykłady znacznie poprawiają wyniki
  • Zapobiegaj halucynacjom — "Jeśli nie wiesz, powiedz to" pomaga (ale nie zawsze)

Podsumowanie

Prompt Engineering to jedna z najtańszych i najskuteczniejszych technik poprawy jakości odpowiedzi modelu. Few-shot, Chain of Thought, ToT i ReAct — te techniki działają na każdym dużym LLM i mogą dać różnice rzędu 30–50% w jakości outputu.

Plusy

  • Darmowe — nie wymaga modyfikacji modelu ani fine-tuningu

Minusy

  • Prompty są trudne do uogólnienia — działa na jednym modelu, niekoniecznie na innym
  • Często wymaga eksperymentowania (A/B testów promptów)
  • Koszt tokenów rośnie z długością promptu (szczególnie few-shot i CoT)
