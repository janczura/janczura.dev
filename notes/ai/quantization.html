Quantization (GPTQ, AWQ)

Kompresja modeli — z FP16 do int4 bez utraty jakości.

Czym jest Quantization?

Quantization to technika zamiany wag modelu z precyzji float32 (lub float16) na niższą — np. int8 lub int4. Zmniejsza to rozmiar modelu i przyspiesza inferencję przy minimalnej utracie jakości. 

# Przykład:
# FP16:     7B params × 2 bajty = ~14 GB VRAM
# INT8:     7B params × 1 bajt  = ~7 GB VRAM
# INT4:     7B params × 0.5 bajta = ~3.5 GB VRAM

GPTQ — Quantization w czasie inference

GPTQ (Grover et al., 2022) to post-training quantization, która kwantyzuje wagі model po treningu. Używa kwantylizacji per-token z optymalizacją na pojedynczym zbiorze walidacyjnym — daje bardzo dobre rezultaty. 

import auto_gptq
from transformers import AutoTokenizer, GPTQForCausalLM

model = GPTQForCausalLM.from_pretrained(
    model_name,
    quantization_config=auto_gptq.AutoQuantizationConfig(wbits=4, group_size=128),
)

AWQ — Activation-Aware Quantization

AWQ (Zuo et al., 2023) to nowsza metoda — bierze pod uwagę rozkład aktywacji (nie tylko wag). Wykrywa "outliers" w aktywacjach i kwantyzuje tylko bezpieczne części. W praktyce: lepsza jakość przy int4 niż GPTQ. 

from awq import AutoAWQForCausalLM, AWQConfig

model = AutoAWQForCausalLM.from_pretrained(model_name)
quant_config = AWQConfig(bits=4, group_size=128)
model.quantize(tokenizer=tokenizer, quant_config=quant_config)
model.save_quantized("./qwen-7b-int4")  # ~3.5 GB zamiast ~14 GB (FP16)

Kiedy kwantyzować?

  • Wdrożenie na GPU z małą pamięcią — 7B model w FP16 = 14GB VRAM, w int4 = ~3.5 GB
  • Inferencja lokalna (MacBook, laptop) — GGUF format z llama.cpp (AWQ-friendly)
  • Serwerowa wydajność — quantized models obsługują więcej żądań na GPU

Podsumowanie

Quantization to najtańszy sposób na "wciśnięcie" dużego modelu na małe GPU. AWQ w int4 daje najlepsze rezultaty — model 7B waży ~3.5 GB i działa płynnie na jednym GPU RTX 3080/4060. GPTQ jest starszy, ale wciąż solidny.

Plusy

  • Mniejszy rozmiar — 4× przy int4 vs FP16 bez fine-tuningu
  • Szybsza inferencja (mniej transferów pamięci)
  • AWQ/GPTQ = post-training — nie trzeba re-trainować modelu

Minusy

  • Jakość spada (zwłaszcza przy int4) — ale AWQ minimalizuje ten spadek
  • Nie można łatwo "odkwantyzować" — model jest trwale skompresowany
  • Kwantyzacja po fine-tuningu (SFT/LoRA) = ryzyko pogorszenia jakości adaptera
